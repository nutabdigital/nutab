## Technologies Used

- **Framework**: [Next.js 14 (App Router)](https://nextjs.org/)
- **Language**: TypeScript + React 18
- **Styling**: Tailwind CSS (via PostCSS)
- **3D & Graphics**: Three.js, @react-three/fiber, @react-three/drei
- **Animations**: Framer Motion, GSAP
- **Icons**: lucide-react, react-icons
- **Deployment**: Static export to [GitHub Pages](https://pages.github.com/)

## Getting Started

### Prerequisites

- Node.js 18.17+ (LTS recommended)
- npm or yarn

---

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/nutabdigital/nutab.git
   cd nutab
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run the development server:
   ```bash
   npm run dev
   ```
4. Open http://localhost:3000 to view the website locally.

---

## Available Scripts

- `npm run dev`: Starts Next.js dev server and opens the app.
- `npm run build`: Builds the production bundle.
- `npm run export`: Exports the site to static HTML in `out/` (required for GitHub Pages).
- `npm run start`: Starts the production server (not used for static hosting).
- `npm run lint`: Runs ESLint checks.

---

### Build for Production

To create a production build:

```bash
npm run build
```

## Deployment

This project is configured for static export and hosting on GitHub Pages.

1. Build and export static files:
   ```bash
   npm run build
   npm run export
   ```
2. The static site is generated in `out/`. Publish this folder to GitHub Pages (e.g., via a `pages` branch or GitHub Actions).
3. A `CNAME` file is included for the custom domain `nutab.ca`.
4. Images are configured as unoptimized for static export (`next.config.mjs`).

## SEO & Analytics

- Metadata: Configured in `app/layout.tsx` using Next.js `Metadata`.
- Open Graph & Twitter: Titles, descriptions, images set for social sharing.
- Structured Data: JSON-LD for Organization/LocalBusiness injected in layout.
- Sitemap & Robots: `public/sitemap.xml` and `public/robots.txt` included.
- Google Analytics: `gtag.js` loaded with delayed page_view and IP anonymization.
- Dark/Light Mode: `data-theme` persisted via `localStorage`; toggle component provided.
